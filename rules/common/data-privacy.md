# Data Privacy in Marketing

## Core Principle

Customer data is a trust asset. Every data practice must be justifiable, transparent, and minimal.

## Data Collection

### Consent Requirements
- Obtain explicit, informed consent before collecting personal data
- Consent must be freely given — no pre-checked boxes, no bundled consent
- Record when and how consent was obtained (consent receipts)
- Separate consent for different purposes (marketing emails vs analytics vs third-party sharing)
- Make it as easy to withdraw consent as it is to give it

### Data Minimization
- Collect only the data you actually need for the stated purpose
- Don't ask for phone numbers if you only send emails
- Don't collect demographics unless you'll actually use them for segmentation
- Regularly review what data you're collecting and purge what's unnecessary

## Data Storage & Processing

### Security
- Never store PII in plain text in marketing tools, spreadsheets, or documents
- Use encryption for data at rest and in transit
- Restrict access to customer data on a need-to-know basis
- Audit who has access to marketing databases regularly

### Third-Party Tools
- Verify that all martech vendors comply with relevant privacy regulations
- Maintain a data processing agreement (DPA) with each vendor
- Know where your data is stored geographically (EU data in EU, etc.)
- Review vendor privacy practices annually

## Data Usage in Marketing

### Personalization
- Personalization should be helpful, not creepy
- Don't reference data the user doesn't know you have
- Avoid hyper-specific targeting that feels invasive
- Give users control over personalization preferences

### Segmentation
- Use anonymized/aggregated data for audience insights
- Don't create segments that could be used for discrimination
- Don't target based on sensitive attributes (health, religion, political views)
- Document segmentation criteria and their business justification

### Customer Data in Content
- NEVER use real customer names/data in case studies without written permission
- Anonymize data in marketing reports and dashboards
- Get explicit approval before sharing customer stories publicly
- Offer review and approval rights on case studies and testimonials

## Data Retention & Deletion

- Define retention periods for each type of marketing data
- Automatically purge data that exceeds retention period
- Honor deletion requests (GDPR right to erasure) within 30 days
- Deletion means deletion — not just deactivation or archiving
- Maintain deletion logs for compliance documentation

## Incident Response

If a data breach occurs:
1. Contain the breach immediately
2. Document what data was affected
3. Notify relevant authorities within 72 hours (GDPR)
4. Notify affected individuals without undue delay
5. Review and fix the root cause
6. Update security measures to prevent recurrence

## Checklist

Before any campaign or data operation:
- [ ] Consent is in place for all data being used
- [ ] Data minimization principle applied
- [ ] Third-party tools have DPAs in place
- [ ] No PII exposed in marketing materials
- [ ] Retention periods defined and enforced
- [ ] Deletion/unsubscribe mechanisms working
- [ ] Team access reviewed and appropriate
